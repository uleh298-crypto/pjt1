# SSABRE 시연 시나리오 (Web + Mobile + 인프라)

## 1. 대상 프로젝트 요약
이 폴더에는 다음 구성요소가 포함됩니다.

- `frontend/`: 웹 프론트엔드 (React + TypeScript + Vite)
- `backend/`: 백엔드 API (Spring Boot, Redis, PostgreSQL)
- `nginx/`: 리버스 프록시 및 정적 파일 서빙
- `android/`: Android 앱
- `iOS/`: iOS 앱
- `monitoring/`: Grafana/Prometheus/Loki/Netdata 모니터링

본 시연은 웹을 기준으로 상세 흐름을 설명하고, 모바일은 동일 기능 흐름을 축약 설명합니다.

## 2. 사전 준비

### 2.1 환경 준비
- 루트에 `.env` 생성 후 DB 정보 입력
- Docker 실행 가능 상태 확인
- 실행: `docker compose up -d --build`

### 2.2 접속 확인
- 웹 프론트: `http://localhost`
- 백엔드 API: `http://localhost/api`
- Grafana: `http://localhost:3001` (admin / `.env`의 `GRAFANA_ADMIN_PASSWORD`)
- Prometheus: `http://localhost/prometheus`
- NetData: `http://localhost/netdata`

### 2.3 시연용 계정/데이터
- 계정은 회원가입 플로우로 생성
- 시연 중 데이터는 직접 작성/생성

## 3. Web 시연 시나리오

### 3.1 랜딩 → 로그인/회원가입
1. **랜딩 화면 진입**
화면: 랜딩 페이지
액션: 상단 CTA `로그인` 또는 `회원가입` 클릭
결과: 로그인/회원가입 화면으로 이동

2. **회원가입 (신규 계정)**
화면: 회원가입 1/2/3단계
액션: 단계별 필드 입력 후 `다음` 진행
결과: `회원가입이 완료되었습니다. 다시 로그인해 주세요.` 알림 확인

3. **로그인**
화면: 로그인 화면
액션: `아이디(이메일)`, `비밀번호` 입력 후 `로그인` 클릭
결과: `/home`으로 이동

### 3.2 홈 화면 기본 동선
4. **헤더/사이드바 확인**
화면: 홈
액션: 상단 헤더 로고 `싸브리타임`, 우측 `알림`(벨), `내 정보` 확인
결과: 기본 내비게이션 위치 확인

5. **사이드바 이동 테스트**
화면: 좌측 사이드바
액션: `홈` → `게시판` → `그룹 찾기` → `나의 그룹` → `쪽지함` 순서로 클릭
결과: 각 페이지 정상 라우팅

### 3.3 커뮤니티(게시판)
6. **게시판 목록 확인**
화면: 게시판 목록
액션: 상단 드롭다운 `전체보기` 클릭, `인기글` 선택
결과: 인기글 필터 적용

7. **게시판 검색**
화면: 게시판 목록
액션: 검색창 `게시판 내 검색...`에 키워드 입력
결과: 목록 실시간 필터링

8. **게시글 작성**
화면: 글쓰기
액션: 우측 상단 `글 쓰기` 클릭
액션: `게시판 선택`, `제목`, `내용` 입력
액션: `사진 첨부`로 이미지 추가, 필요 시 `투표 추가`
액션: 하단 `등록하기` 클릭
결과: `게시글이 등록되었습니다.` 알림, 목록으로 복귀

9. **게시글 상세 → 채팅 유도**
화면: 게시글 상세
액션: 게시글 클릭 후 상세 진입
액션: `채팅하기` 버튼 클릭
결과: 상대와 새 채팅방 생성 (또는 실패 시 오류 알림)

### 3.4 그룹 찾기 → 신청
10. **그룹 찾기 탭 전환**
화면: 그룹 찾기
액션: 탭 `스터디` ↔ `프로젝트` 클릭
결과: 목록이 각 유형으로 전환

11. **그룹 검색/필터**
화면: 그룹 찾기
액션: 검색창에 키워드 입력, `필터` 칩 선택
결과: 필터/검색 반영된 리스트 확인

12. **그룹 상세 확인**
화면: 그룹 상세
액션: 카드 클릭
결과: 상세 정보(기간, 인원, 설명, 리더/멤버) 표시

13. **그룹 신청**
화면: 그룹 상세 하단
액션: `신청하기` 클릭
결과: 신청 폼으로 이동

14. **신청서 작성**
화면: 그룹 신청
액션: `포트폴리오` 선택, `지원 제목`, `포지션`, `지원 메시지` 입력
액션: 하단 `신청` 클릭
결과: `신청이 완료되었습니다!` 알림, 상세로 복귀

### 3.5 나의 그룹 관리 (리더/멤버)
15. **나의 그룹 목록**
화면: `나의 그룹`
액션: `스터디/프로젝트` 탭 전환, 카드 클릭
결과: 그룹 상세(관리용) 화면 이동

16. **공지사항 확인**
화면: 그룹 상세(관리용) 내 `공지사항`
액션: 공지 리스트 확인
결과: 공지 유무 확인 (비어있을 경우 안내 문구 표시)

17. **일정 추가**
화면: 그룹 상세(관리용)
액션: 상단 `일정 추가` 버튼(아이콘) 클릭
액션: `제목`, `내용`, `시작일`, `종료일`, `상태` 입력
액션: `추가` 클릭
결과: 일정 카드 생성

18. **일정 상태 변경**
화면: 일정 카드
액션: `예정/진행중/완료` 상태 버튼 클릭
결과: 상태 뱃지 및 버튼 비활성 상태 업데이트

19. **멤버 관리**
화면: 우측 상단 메뉴 `⋮` 클릭
액션: `멤버 관리` 선택
결과: 멤버 관리 화면으로 이동

### 3.6 메시지(채팅)
20. **채팅 목록 확인**
화면: `쪽지함`
액션: 최근 메시지 카드 클릭
결과: 채팅 상세 화면 이동

21. **메시지 송수신**
화면: 채팅 상세
액션: 메시지 입력 후 전송
결과: 대화 리스트에 새 메시지 반영

### 3.7 알림/설정
22. **알림 목록 확인**
화면: 상단 헤더 `알림` 아이콘
액션: 클릭
결과: 알림 리스트 화면 이동

23. **입퇴실 알림 토글**
화면: `내 정보` → `설정`
액션: `🔔 입퇴실 알림` ON/OFF 버튼 클릭
결과: 활성화/비활성화 알림 메시지 표시

### 3.8 내 정보/포트폴리오
24. **내 정보 화면**
화면: `내 정보`
액션: 프로필 이미지 클릭, `설정`, `로그아웃` 버튼 확인
결과: 프로필 상세/로그아웃 다이얼로그 확인

25. **포트폴리오 작성**
화면: `내 정보` → `포트폴리오` → `새 포트폴리오 작성`
액션: `포트폴리오 제목`, `한 줄 소개`, `자기소개` 입력
액션: `기술 스택` 검색 및 추가, 숙련도 선택
액션: `백준 아이디 연동` → `연동 확인` 클릭
액션: `관련 링크` 추가 후 `저장하기`
결과: 포트폴리오 상세 화면으로 이동

26. **프로젝트 작성 화면 확인**
화면: `/portfolio/project/:portfolioId`
액션: 해당 경로 진입
결과: 현재는 `Project Write` 간단 페이지 표시 (향후 확장 포인트로 소개)

## 4. Mobile 앱 시연 시나리오 (Android/iOS 공통)

1. **앱 실행 → 로그인**
화면: 로그인
액션: 계정 입력 후 로그인
결과: 메인 화면 진입

2. **그룹 찾기 → 상세 → 신청**
화면: 그룹 리스트
액션: `스터디/프로젝트` 전환, 카드 선택
결과: 상세 화면 → `신청하기` 버튼으로 신청

3. **커뮤니티 → 게시글 작성**
화면: 게시판
액션: `글쓰기` 클릭, 제목/내용 입력, 등록
결과: 목록에 신규 글 반영

4. **채팅 → 실시간 메시지**
화면: 메시지 리스트
액션: 채팅방 진입 후 메시지 전송
결과: 실시간 수신 확인

5. **알림 수신**
화면: 알림/푸시
액션: 앱 푸시 알림 수신 확인
결과: 알림 상세 화면으로 이동

## 5. 인프라/모니터링 시연 (선택)

1. **Grafana 대시보드**
화면: `http://localhost:3001`
액션: admin / `.env`의 `GRAFANA_ADMIN_PASSWORD`로 로그인
결과: 서비스 메트릭 시각화

2. **Prometheus/NetData**
화면: `http://localhost/prometheus`, `http://localhost/netdata`
액션: 리소스 메트릭 및 서버 상태 확인
결과: CPU/메모리/네트워크/로그 흐름 확인

## 6. 시연 팁
- 데이터가 없으면 화면이 빈 상태로 보일 수 있으므로, 시연 시작 전 최소 1개 게시글/그룹/포트폴리오를 미리 생성
- 그룹 신청/멤버 관리는 리더 권한이 필요한 경우가 있으므로 리더 계정 준비 권장
- 알림 시연은 브라우저 알림 권한 허용이 필요
